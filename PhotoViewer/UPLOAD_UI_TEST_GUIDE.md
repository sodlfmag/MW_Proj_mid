# Upload UI 테스트 및 캡처 가이드

## 구현된 Upload UI 확인

### 구현 상태
✅ **Post에 따른 UI 제공 방식** (상 등급)
- 이미지 선택 기능
- 제목/내용 입력 다이얼로그
- 서버 업로드 기능

## 테스트 및 캡처 절차

### 1단계: 앱 빌드 및 실행
1. Android Studio에서 프로젝트 열기
2. 에뮬레이터 또는 실제 기기 연결
3. 앱 빌드 및 실행
4. **캡처**: 앱 시작 화면 (동기화/업로드 버튼이 보이는 화면)

### 2단계: Upload UI 확인
1. **"새로운 이미지 게시" 버튼 클릭**
2. 갤러리/이미지 선택 화면 표시
3. 이미지 선택
4. **캡처 대상**: 다이얼로그 화면 (제목/내용 입력 UI)
   - 이미지 미리보기
   - 제목 입력 필드
   - 내용 입력 필드
   - 업로드/취소 버튼

### 3단계: 업로드 실행
1. 제목 입력 (예: "테스트 게시물")
2. 내용 입력 (예: "Upload UI 테스트입니다")
3. **"업로드" 버튼 클릭**
4. **캡처**: 업로드 진행 중 또는 완료 메시지

### 4단계: 업로드 결과 확인
1. 서버에서 게시물 확인
   - 브라우저에서 `http://127.0.0.1:8000/api_root/Post/` 접속
   - 또는 `http://127.0.0.1:8000/` 접속하여 게시물 목록 확인
2. **캡처**: 업로드된 게시물이 서버에 표시되는 화면

## 제출용 캡처 화면

### 필수 캡처
1. **다이얼로그 UI 화면** (Post에 따른 UI 제공)
   - 이미지 미리보기
   - 제목 입력 필드
   - 내용 입력 필드
   - 업로드/취소 버튼이 모두 보이는 화면

2. **업로드 완료 메시지**
   - Toast 메시지 "Upload 성공!" 화면

### 선택 캡처
3. **서버에 업로드된 게시물 확인**
   - 브라우저에서 게시물 목록 또는 상세 페이지

## 주의사항

### 서버 실행 확인
테스트 전에 Django 서버가 실행 중이어야 합니다:
```bash
cd PhotoBlogServer/djangoenv
.\venv\Scripts\Activate.ps1
python manage.py runserver
```

### 네트워크 설정
- **에뮬레이터**: `http://10.0.2.2:8000` (자동 설정됨)
- **실제 기기**: `http://[PC의 IP주소]:8000`으로 변경 필요
  - MainActivity.java의 `site_url` 변수 수정

### 권한 확인
앱 실행 시 저장소 접근 권한을 승인해야 합니다.

## MainActivity 코드 위치

### 주요 메서드
- `onClickUpload(View v)`: 업로드 버튼 클릭 시 이미지 선택
- `onActivityResult()`: 이미지 선택 후 처리
- `showUploadDialog()`: 제목/내용 입력 다이얼로그 표시
- `PutPost AsyncTask`: 서버에 업로드

### UI 레이아웃 파일
- `res/layout/dialog_upload.xml`: 업로드 다이얼로그 UI
